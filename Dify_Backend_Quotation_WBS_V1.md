# Dify 报价 WBS（12 个报价包 / 人天拆分版）

> 估算口径：**中高级全栈/后端为主的人天**，已包含接口、表结构、基础联调、自测，不含独立 UI 设计、压测、云资源费用。  
> 适用场景：老板快速报价、项目经理拆排期、开发拆任务。  
> 估算以当前 Dify 功能广度为目标，按“做出同等级可用系统”而不是“1:1 克隆每个边角功能”估算。

---

## 1. 估算前提

1. 1 人天按 1 名中高级工程师净产出计算。
2. 若要做到与官方 Dify 完全同深度的兼容性、迁移能力、插件生态细节，需在本表基础上再追加。
3. 本表以当前文档里已识别出的主模块为准。

## 4. 建议报价分层

### 4.1 最小可上线版（约 92~110 人天）

包含：

1. 包 01 账号与工作区
2. 包 02 应用中心
3. 包 03 AppModelConfig 基础（不含复杂 Agent）
4. 包 04 会话消息
5. 包 05 Workflow 基础（不含复杂节点）
6. 包 06 Dataset 基础
7. 包 07 文档导入基础
8. 包 08 Provider 凭证

### 4.2 标准商业版（约 150~180 人天）

在最小版基础上增加：

1. 完整 Workflow 调试/历史/归档
2. 完整 metadata / summary / 文档状态机
3. Tool / Trigger / Endpoint / MCP
4. Explore 基础能力

### 4.3 对标 Dify 高配版（约 200~240 人天）

在标准版基础上增加：

1. 完整 Plugin 生命周期
2. 自动升级、动态参数 options
3. 更完整的 runtime / installed-app / explore 生态
4. 更完整的运维、审计、品牌化、导出和边缘接口

---

## 5. 商务说明建议

报价时建议单列以下不含项：

1. UI/视觉设计
2. 云资源费用（模型、对象存储、向量库、队列、日志）
3. 第三方付费接口费用
4. 压测与高可用集群部署
5. 插件生态内容制作
6. 旧系统数据迁移
