# AI Agent 技术演进时间线

## 1. Prompt Engineering（提示工程）
- **起源时间**：2020年左右
- **背景**：随着GPT-3等大型语言模型的出现而兴起
- **核心概念**：通过精心设计的输入提示来引导AI模型产生期望输出
- **特点**：主要依赖于对模型输入的巧妙构造，不需要修改模型本身

## 2. Workflow（工作流）
- **起源时间**：2022-2023年
- **背景**：在Prompt Engineering基础上，为了实现更复杂的多步骤任务自动化
- **核心概念**：将多个AI调用、决策节点和外部工具按照逻辑顺序连接成流程
- **特点**：使AI能够执行需要多个步骤和条件判断的复杂任务

## 3. Skills（技能系统）
- **起源时间**：2023年
- **背景**：为了让AI代理具备更多实用功能而引入的模块化能力扩展机制
- **核心概念**：预定义的功能模块，让AI可以调用特定工具或服务来完成任务
- **特点**：提高了AI的实用性，使其能执行文件操作、网络请求、数据库查询等操作

## 4. Context Engineering（上下文工程）
- **起源时间**：2023-2024年
- **背景**：随着AI代理需要处理更长、更复杂的交互过程而发展起来
- **核心概念**：管理和优化AI模型的上下文信息，以保持长期对话和任务状态
- **特点**：关注如何有效地保留和传递信息，在长时间运行的代理中尤为重要

## 5. Harness Engineering（框架工程）
- **起源时间**：2024年至今
- **背景**：随着AI代理平台和开发框架的成熟而出现的概念
- **核心概念**：设计和构建AI代理的底层架构和运行环境
- **特点**：包括资源管理、安全控制、多代理协调、持久化存储等系统性考虑

---
这份时间线展示了AI代理技术从简单的提示工程逐步发展为完整的系统工程的过程。
